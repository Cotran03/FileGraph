# FileGraph 사용법

이 문서는 FileGraph를 실행하고 수동 그래프 MVP를 써보는 방법을 정리합니다.

## 실행

개발 환경에서는 저장소 루트에서 다음 명령을 실행합니다.

```powershell
.venv\Scripts\python.exe main.py
```

## 테스트

```powershell
.venv\Scripts\python.exe -m pytest
```

## 설정과 개인 데이터

- 개발 환경의 기본 DB는 `db/database.db`를 사용합니다.
- 빌드된 실행 파일은 `%LOCALAPPDATA%/FileGraph/` 아래에 DB와 런타임 데이터를 둡니다.
- UI 설정은 FileGraph DB의 `settings` 테이블에 저장됩니다.
- API 키 원문은 DB나 설정 파일에 저장하지 않고 `keyring`을 사용합니다.

## 기본 흐름

### 1. 파일 또는 폴더 추가

- 오른쪽 작업 패널에서 `파일 추가` 또는 `폴더 추가`를 누릅니다.
- 파일이나 폴더를 그래프 화면에 드래그 앤 드롭해도 됩니다.
- 폴더를 추가할 때는 하위 파일 자동 등록이 기본으로 켜져 있습니다.
- 추가 전 확인 창에서 등록될 폴더/파일/포함 관계 개수를 미리 볼 수 있습니다.
- 하위 파일 등록을 원하지 않으면 확인 창의 `내부 파일도 등록` 체크를 끕니다.
- 내부 파일을 함께 등록하면 폴더에서 각 하위 폴더/파일로 `포함` 관계가 자동 생성됩니다.
- 같은 경로가 이미 등록되어 있으면 기존 노드로 이동하거나 관계 추가 흐름으로 이어갈 수 있습니다.
- `샘플` 버튼으로 예시 노드와 관계를 빠르게 만들 수 있습니다.

### 2. 관계 추가

- `관계 추가` 버튼을 누릅니다.
- 출발 노드와 도착 노드를 선택합니다.
- 관계 타입, 방향성, 관계 강도, 설명을 입력합니다.
- 기본 관계 타입은 `관련 있음`, `참고자료`, `원본에서 생성됨`, `포함`, `다른 버전`입니다.
- 같은 두 노드 사이에 같은 관계 타입을 중복으로 만들면 기존 관계 수정을 안내합니다.

### 3. 그래프 보기와 정리

- 노드를 드래그하면 위치가 저장됩니다.
- `자동 정렬`은 저장된 수동 좌표를 초기화하고 그래프를 다시 배치합니다.
- 오른쪽 상단 `설정` 버튼을 눌러 노드/간선 라벨 표시 방식과 글자 크기를 조절할 수 있습니다.
- 노드 라벨은 `폴더 이름만`, `파일 이름만`, `둘 다`, `마우스가 올라가면 보이기` 중에서 고릅니다.
- `폴더 이름만`에서는 폴더 이름이 기본 표시되고 파일 이름은 마우스를 올리면 보입니다. `파일 이름만`은 반대로 동작합니다.
- `포함` 관계가 있는 폴더 노드는 우클릭 메뉴에서 내부 파일 노드를 접거나 펼칠 수 있습니다.
- 접힌 폴더는 점선 테두리와 포함 파일 개수 배지로 열린 폴더와 구분됩니다.
- 최상위 폴더는 일반 하위 폴더와 다른 테두리/배경으로 구분됩니다.
- 노드를 우클릭하면 열기, 관계 추가, 관계 수정, 접기/펼치기, 강조, 메모, 관계 색상, 노드 삭제 메뉴를 사용할 수 있습니다.
- 오른쪽 드래그로 여러 노드를 선택할 수 있고, 선택된 노드는 함께 이동하거나 삭제할 수 있습니다.
- 노드를 더블클릭하면 파일 또는 폴더를 기본 프로그램으로 엽니다.

### 4. 검색과 포커스 뷰

- 작업 패널 검색창에서 파일명과 경로를 검색합니다. DB에 카테고리나 AI 컨텍스트가 저장된 노드도 검색 대상에 포함됩니다.
- 검색어를 입력하면 검색창 아래에 매칭 후보가 표시되고, 후보를 선택하면 해당 이름으로 검색/포커스할 수 있습니다.
- 노드를 선택한 뒤 단계 값을 조절하면 해당 노드 기준 포커스 그래프를 볼 수 있습니다.
- `전체 보기` 버튼으로 전체 그래프 보기로 돌아갑니다.

### 5. 관계 수정과 삭제

- 오른쪽 관계 목록에서 관계를 선택한 뒤 `수정` 또는 `삭제`를 누릅니다.
- 관계 목록 항목을 더블클릭해도 관계 수정 창을 열 수 있습니다.
- 노드 우클릭 메뉴의 `관계 수정` 하위 메뉴에서도 연결된 관계를 바로 수정할 수 있습니다.
- 관계 삭제는 선택한 관계 레코드를 DB에서 삭제합니다.

### 6. 노드 삭제

- 노드 삭제는 원본 파일이나 폴더를 삭제하지 않습니다.
- FileGraph 안에서만 `DELETED` 상태로 숨깁니다.
- 연결된 관계 기록은 보존되지만 기본 그래프에서는 숨겨집니다.
- 여러 노드를 선택한 상태에서 작업 패널의 삭제 버튼을 누르면 선택된 노드들을 함께 숨깁니다.
- `Ctrl+Z`로 노드 삭제, 관계 삭제, 자동 정렬을 되돌릴 수 있습니다.

### 7. 파일 위치 갱신

- `파일 위치 갱신`을 누르면 등록된 파일/폴더의 현재 경로를 검사하고 변경 감지 정보를 갱신합니다.
- 경로가 정상 접근 가능하면 `ACTIVE`로 둡니다.
- 경로를 찾을 수 없으면 `MISSING`으로 표시합니다.
- 권한 문제로 접근할 수 없으면 `ACCESS_DENIED`로 표시합니다.
- 활성 파일은 SHA-256 해시를 저장합니다. 100MB 초과 파일은 자동 해시 계산에서 제외합니다.
- 앱 시작 시에는 전체 파일 위치 갱신을 즉시 실행하지 않습니다. 대규모 DB에서 시작 자원 사용량을 줄이기 위한 정책입니다.

### 8. 누락 파일 찾기

- `누락 파일 찾기`를 누르고 검색할 폴더를 선택합니다.
- FileGraph는 선택한 폴더 안에서 저장된 해시와 같은 파일을 찾습니다.
- 같은 해시의 파일을 찾으면 기존 `MISSING` 노드의 경로를 새 위치로 복구합니다.
- `.git`, `.venv`, `node_modules`, `__pycache__` 같은 폴더는 검색에서 건너뜁니다. 설정 화면에서 사용자 지정 무시 폴더를 추가할 수 있습니다.

### 9. 설정 화면

- 오른쪽 작업 패널 상단의 `설정` 버튼을 누릅니다.
- 그래프 라벨 글자 크기, 노드 이름 표시 방식, 관계 이름 표시 방식을 설정합니다.
- 파일/폴더 노드 기본 색상과 강조 색상 슬롯을 설정합니다.
- 확장자 아이콘 표에서 `proto -> 코드`, `bak -> 압축 파일`처럼 사용자 지정 매핑을 추가할 수 있습니다. 사용자 지정 매핑은 기본 확장자 매핑보다 우선합니다.
- 무시 폴더 목록을 쉼표로 구분해 관리합니다.
- AI 설정 영역은 dev-ai에서 OpenRouter/직접 연결 방식, 공급자/모델, API 키 저장/삭제를 다루도록 확장할 예정입니다.

### 10. 단축키

- `Ctrl+F`: 검색창으로 이동
- `Esc`: 검색 또는 선택 해제
- `Delete`: 선택 노드 삭제
- `Enter`: 선택 노드 열기
- `Ctrl+L`: 자동 정렬
- `Ctrl+R`: 그래프 새로고침
- `Ctrl+Shift+R`: 파일 위치 갱신
- `F`: 선택 노드 포커스 보기
- `Shift+F`: 전체 보기
- `Ctrl+N`: 파일 추가
- `Ctrl+Shift+N`: 폴더 추가
- `Ctrl+Z`: 되돌리기
- `F1`: 단축키 도움말
- `Ctrl+Shift+L`: 범례 보기
- `Ctrl+Shift+O`: 고립 노드 보기
- `Ctrl+Shift+D`: 중복 후보 보기
- `Ctrl+Shift+B`: DB 백업
- `Ctrl+Shift+E`: JSON 내보내기

## 현재 한계

- AI 관계 추천은 아직 구현되지 않았습니다.
- AI 폴더 정리와 실제 파일 이동 자동화는 아직 구현되지 않았습니다.
- 이동 Undo는 아직 구현되지 않았습니다.
- 관계 타입을 직접 추가/수정하는 전용 화면은 아직 구현되지 않았습니다.
- 네트워크 드라이브는 공식 지원 범위가 아닙니다.
- 대규모 DB 기준 성능 QA와 장시간 백그라운드 작업 취소 UX는 추가 보강 대상입니다.
