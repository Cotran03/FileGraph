# FileGraph

FileGraph는 파일 시스템의 물리적인 폴더 구조와 별개로 파일과 폴더 사이의 논리적 관계를 저장하고 시각화하는 Windows 데스크톱 앱입니다.

파일을 어디에 넣어두었는지보다 "이 파일이 무엇과 연결되어 있는지"를 중심으로 관리합니다. 사용자는 파일과 폴더를 노드로 등록하고, 노드 사이에 관계 타입, 방향성, 강도, 설명을 가진 간선을 만들 수 있습니다.

## 핵심 아이디어

- 파일과 폴더를 그래프의 `Nodes`로 관리한다.
- 파일-파일, 파일-폴더, 폴더-폴더 사이의 논리적 연결을 `Relations`로 저장한다.
- 관계 강도에 따라 그래프 배치와 간선 표현을 다르게 보여준다.
- 노드는 폴더, 파일 확장자, 누락/권한 상태에 맞는 아이콘으로 구분한다.
- 사용자가 드래그한 노드 위치는 저장되어 다음 실행 때도 유지된다.
- 파일이 사라지거나 접근할 수 없어도 관계 기록은 바로 지우지 않고 상태값으로 보존한다.
- 이후 버전에서는 AI가 관계를 제안하고, 사용자가 승인한 규칙에 따라 폴더 정리를 돕는다.

## 현재 상태

수동 그래프 관리 MVP는 구현되어 있습니다. 파일/폴더 노드 등록, 폴더 내부 파일 일괄 등록과 포함 관계 자동 생성, 관계 추가/수정/삭제, 그래프 표시, 확장자/상태별 아이콘 표시, 검색, 포커스 뷰, 좌표 저장, 파일 상태 확인, 해시 기반 누락 파일 복구를 사용할 수 있습니다.

MVP에 더해 파일 무결성 기능 일부도 들어가 있습니다. 앱은 파일 경로를 검사해 `ACTIVE`, `MISSING`, `ACCESS_DENIED` 상태를 반영하고, 저장된 SHA-256 해시를 이용해 선택한 폴더 안에서 누락 파일을 다시 찾을 수 있습니다.

아직 구현 전인 큰 기능은 AI 관계 추천, AI 폴더 정리, 실제 파일 이동 자동화, 이동 Undo입니다.

## 목표 사용자

FileGraph는 폴더 정리 규칙보다 파일 사이의 의미 관계가 더 중요한 사람을 위한 도구입니다.

- 프로젝트 파일이 여러 폴더에 흩어져 있어 맥락을 잃기 쉬운 경우
- 기획서, 발표자료, 예산표, 디자인 소스처럼 서로 참조하는 파일을 함께 보고 싶은 경우
- 물리적 폴더 구조를 자주 바꾸지 않고도 파일 관계를 기록하고 싶은 경우
- 나중에 AI 정리 기능을 붙이기 전에 수동으로 안전하게 관계 데이터를 쌓고 싶은 경우

## 개발 방향

1차 목표는 AI 없이도 동작하는 수동 파일 관계 그래프입니다. 이 범위는 현재 충족된 상태입니다.

다음 단계는 관계 타입 관리, 더 정교한 파일 이동 추적, AI 관계 추천, AI 폴더 정리 순서로 확장하는 것입니다. AI 기능은 외부 전송 동의, 사용자 API 키, 미리보기, 승인 절차를 전제로 보조 기능으로 붙입니다.
